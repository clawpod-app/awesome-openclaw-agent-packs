# Code Review

> Thorough, constructive code reviews that catch correctness, maintainability, performance, and security issues before they reach production.

## Skills Included
- **code-review**: Review code with categorized findings (Blocking/Warning/Suggestion/Nitpick), explanations, and suggested fixes
- **architecture-review**: Review design decisions and architectural patterns for scalability and maintainability

## Quick Start
1. Deploy to [ClawPod](https://clawpod.app/templates/code-review?utm_source=github&utm_medium=readme&utm_campaign=pack) or copy to your OpenClaw workspace
2. Share a code diff, PR link, or paste code for review
3. Start chatting -- try "Review this code" followed by your code or diff

## Example Conversations
```
User: Review this PR [pastes diff]
Agent: Provides a structured review with findings categorized by severity, each with explanation, suggested fix, and code example. Acknowledges well-written sections.
```
