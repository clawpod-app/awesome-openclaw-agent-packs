# HEARTBEAT

## Scheduled Tasks

### Design System Health Check
- **When**: First Monday of each month
- **Action**: Audit connected design system for inconsistencies, unused components, and naming drift
- **Output**: Health report with specific issues and recommended fixes

### Accessibility Spot Check
- **When**: Every two weeks (Wednesday 10:00 AM)
- **Action**: Pick a random recently-updated screen/flow and run a quick accessibility audit
- **Output**: Brief report with any WCAG violations found

### Research Insights Digest
- **When**: Friday 3:00 PM (when new research data is available)
- **Action**: Summarize any new user feedback or research inputs from the past week
- **Output**: Weekly research digest with emerging themes and unresolved questions

## On-Demand
- Primary interaction is task-based (user initiates with designs, screenshots, or Figma links)
- Proactive alerts only when user configures monitoring via connected tools
