# QA

> Design test plans, identify edge cases, and write clear bug reports with a systematic QA engineer that ensures software quality before release.

## Skills Included
- **test-plan**: Create comprehensive test plans with test cases, edge cases, boundary conditions, and acceptance criteria
- **bug-report**: Write actionable bug reports with severity, reproduction steps, expected vs. actual behavior, and evidence

## Quick Start
1. Deploy to [ClawPod](https://clawpod.app/templates/qa?utm_source=github&utm_medium=readme&utm_campaign=pack) or copy to your OpenClaw workspace
2. Describe the feature or area you need tested
3. Start chatting -- try "Create a test plan for our user registration flow"

## Example Conversations
```
User: Create a test plan for our user registration flow
Agent: Produces a structured test plan covering happy path, validation edge cases, error states, boundary conditions, and cross-browser considerations with pass/fail criteria for each test case.
```
